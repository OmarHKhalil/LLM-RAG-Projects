# Venezia Bank Finance Chatbot

A production-oriented Retrieval-Augmented Generation (RAG) chatbot for banking and financial assistance. This repository contains a research notebook for prototyping the pipeline and a FastAPI service for serving the chatbot as an API.

The system is designed to answer finance-related questions in a grounded, domain-safe, and professional manner while preserving the identity of Venezia Bank.

## 1. Project Overview

This project combines:

- a local FAQ knowledge base derived from banking documents,
- semantic embeddings generated by the all-MiniLM-L6-v2 model,
- persistent vector search powered by ChromaDB, and
- response generation through Gemini 2.5 Flash with an Ollama fallback path.

The goal is to provide accurate, context-aware financial responses without relying on static prompt-only generation.

## 2. Key Features

- Context-grounded responses using a local vector database
- Domain restriction to finance and banking topics
- Venezia Bank persona and professional tone
- Greeting and conversational support
- Persistent knowledge base that can be updated without retraining the model
- FastAPI endpoint for integration into web or mobile applications

## 3. Architecture and RAG Pipeline

The solution follows a standard retrieval-augmented workflow:

| Stage                | Component           | Implementation                                      | Purpose                                                          |
| -------------------- | ------------------- | --------------------------------------------------- | ---------------------------------------------------------------- |
| Data Preparation     | FAQ Dataset         | BankFAQs2.csv and BankFAQs1.csv                     | Store banking FAQs and answers in a structured format            |
| Document Engineering | LangChain Documents | Question + Answer merged into a single content unit | Create retrievable knowledge chunks                              |
| Embedding            | all-MiniLM-L6-v2    | HuggingFaceEmbeddings                               | Convert text into semantic vectors                               |
| Vector Storage       | ChromaDB            | Chroma collection persisted to disk                 | Store embeddings and enable similarity search                    |
| Retrieval            | Retriever           | Chroma as_retriever with k = 1                      | Return the single most relevant context document                 |
| Prompting            | PromptTemplate      | System instructions for finance/banking behavior    | Guide the model to answer safely and consistently                |
| Generation           | Gemini 2.5 Flash    | ChatGoogleGenerativeAI                              | Produce the final polished answer                                |
| Fallback             | Ollama              | ChatOllama                                          | Provide a local fallback when Gemini credentials are unavailable |
| Delivery             | FastAPI             | POST endpoints                                      | Expose the chatbot as an HTTP service                            |

### RAG Flow

1. The user submits a natural-language banking question.
2. The retriever searches the ChromaDB vector store for the most relevant passage.
3. The retrieved context is injected into a structured prompt.
4. The LLM generates a grounded answer using the retrieved information.
5. The response is returned as text or HTML through the API.

## 5. Data Preprocessing and Knowledge Base Construction

The FAQ dataset was adapted to the Venezia Bank context by:

- starting from a banking FAQ dataset originally associated with the Indian bank HDFC,
- replacing the original bank name with Venezia Bank,
- converting currency references from Indian rupees to Syrian pounds,
- aligning the content with the target banking domain,
- enriching the dataset with additional questions and answers specific to our bank and application,
- adding domain-relevant content for common financial and banking scenarios such as loans, accounts, cards, and security.

The resulting records were packaged into LangChain documents and indexed into ChromaDB for semantic retrieval.

## 7. Setup and Run

### Prerequisites

- Python 3.10+
- Optional: a valid Gemini API key for direct Gemini usage
- Optional: Ollama installed locally for fallback generation

### Installation

```bash
cd genai-finance-chatbot-final-API
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

### Configure the API

The service in app.py uses:

- a local embedding model directory named path_to_local_model,
- a persisted Chroma vector store under vectorstore,
- a Gemini model named gemini-2.5-flash,
- and an Ollama fallback using gemma4:e2b.

If you want to use Gemini directly, add your API key in the GOOGLE_API_KEYS list inside app.py.

### Start the API Server

```bash
uvicorn app:app --host 0.0.0.0 --port 8000 --reload
```

### Example API Requests

#### JSON Response

```bash
curl -X POST "http://127.0.0.1:8000/genai-finance-chatbot" \
  -H "Content-Type: application/json" \
  -d '{"question":"What types of loans are available?"}'
```

#### HTML Response

```bash
curl -X POST "http://127.0.0.1:8000/genai-finance-chatbot_html" \
  -H "Content-Type: application/json" \
  -d '{"question":"How can I open an account?"}'
```

### Notebook Workflow

To explore the prototype interactively:

```bash
jupyter notebook final-genai-finance-chatbot.ipynb
```

Run the notebook cells sequentially to build the vector store and test the retrieval pipeline.

## 9. Test Scenarios

The system was tested across several types of user inputs:

| Scenario                                | User Input                         | Expected Behavior                                                                    |
| --------------------------------------- | ---------------------------------- | ------------------------------------------------------------------------------------ |
| In-Scope Query (Stored)                 | What types of loans are available? | Retrieve the relevant FAQ content and respond in the Venezia Bank persona            |
| In-Scope Query (Unseen Financial Query) | What are financial contracts?      | Provide a coherent finance-related explanation based on the retrieved context        |
| Out-of-Scope Query                      | Who won the World Cup?             | Reject the request and state that only financial and banking questions are supported |
| Greeting                                | Hi, how are you?                   | Respond politely without triggering retrieval                                        |

## 10. Evaluation Metrics

The chatbot was evaluated using 300 banking-related questions stored in the knowledge base.

| Metric              | Result |
| ------------------- | -----: |
| Semantic Similarity | 83.27% |
| BERTScore (F1)      | 88.29% |

These results indicate strong semantic preservation and high-quality answer generation while maintaining a professional banking tone.
